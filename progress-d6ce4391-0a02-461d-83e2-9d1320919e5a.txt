## Codebase Patterns
- All CLI tests use isolated temp HOME directories via `fs.mkdtempSync` and `spawn` with `HOME` env override
- CLI script path: `dist/cli/cli.js` (built from `src/cli/cli.ts`)
- Workflow YAML specs are loaded via `loadWorkflowSpec()` from `src/installer/workflow-spec.js`
- Bundled workflow directories are resolved via `resolveBundledWorkflowDir()` from `src/installer/paths.js`
- `getWorkflowShortDescription()` extracts the first sentence from a workflow's description field
- `listBundledWorkflows()` returns sorted directory names under the bundled workflows dir
- CLI command handlers follow the pattern: `if (group === "workflow") { if (action === "list") { ... } }`
- New imports go in the import block at the top of `src/cli/cli.ts`
- Dispatcher AGENTS.md is a workflow persona file at `workflows/just-do-it/agents/dispatcher/AGENTS.md`
- New workflows (do-now, do-review-do-verify) are standalone, non-family workflows
- Workflow YAML `|` block scalars preserve content as-is; `readFileSync` returns raw text including all indentation
- `assert.match` regex `.` does NOT match newlines by default; use `[\s\S]` for cross-line patterns
