## Codebase Patterns
- CLI help infrastructure lives in `src/cli/cli.ts` with three shared functions: `hasHelpFlag`, `printHelp`, `printHelpSubcommand`
- Per-command help functions follow `get{Group}{Action}Help()` naming convention (e.g., `getStepPeekHelp`, `getMcpStartHelp`)
- --help dispatch sits at the very top of `main()` before any side effects
- `getUsageText()` opens with "Run tamandua <command> --help for detailed command help."
- Tests use Node's built-in `node:test` and `node:assert`, run with `npm test`
- Build with `npm run build` (tsc + HTML copy + version injection)
