## Codebase Patterns
- Workflow specs live in `workflows/<id>/workflow.yml`, parsed via `loadWorkflowSpec()`
- Step input templates contain both consumed `{{placeholders}}` and output-contract `KEY: value` lines
- Step `expects` field uses `regex:^KEY:` patterns to enforce output format at parseOutputKeyValues level
- `collectPlaceholders()` uses `/{{(\w+(?:\.\w+)*)}}/g` — identical in simulation test and contract lint
- AUTO_CONTEXT_KEYS and HARNESS_SEEDED_CONTEXT_KEYS must stay in sync between contract lint and simulation tests
- Tests import from `dist/` (built output), not `src/` — always run `npm run build` before testing
- RESERVED_CONTEXT_KEYS in step-ops.ts defines keys that cannot be overwritten by step output
- Expects strings contain literal `\n` (YAML escape) — split on `\n` not `\\n`
- 15 branch-producing workflows have `regex:^BRANCH:\s*\S+` in step 0 expects; 8 others don't
- `regex:PR:` (no caret) in PR-producing steps for github-pr workflow variants
- Shell command detection: backtick-quoted commands or lines with common shell words (git, npm, cd, etc.)
- Quarantine workflows are the only legitimate exception to stricter tier: no triage/plan step, so no regex:^BRANCH: producer exists
